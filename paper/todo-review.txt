---
Peer Review Report

Manuscript: "Quantitative Analysis of the Impact of Transform Ordering and
Selection in Image Preprocessing for Skin Disease Classification using
Convolutional Neural Networks"

Authors: Tkacz, G.M. and Sampaio, G.S.

Venue: IEEE Transactions on Image Processing

---
Summary Statement

This paper presents an exhaustive combinatorial evaluation of 65 preprocessing
  pipelines (all permutations of four image transforms at lengths 0–4) applied
to a CNN for binary skin lesion classification. The central finding — that
transform ordering dominates selection as the primary performance determinant
for pipelines of three or more stages — is novel, well-supported by the data,
and practically useful. The variance decomposition, permutation-validated
positional analysis, and confusion-matrix error decomposition are
methodologically sound and internally consistent.

Overall Recommendation: Minor to Moderate Revisions

Key Strengths:
- Genuinely novel research question: no prior work has exhaustively mapped the
  ordering-vs-selection landscape for preprocessing pipelines
- Complete combinatorial design eliminates cherry-picking bias; all 65
configurations are evaluated
- Strong internal consistency: every numerical claim in the prose matches the
results table (independently verified)
- Clear, actionable practical template (E → {D,CS} → N) derived from empirical
  evidence

Key Weaknesses:
- Single CNN architecture and single binary task severely limits
generalizability claims
- Loss function choice (MSE for binary classification) is non-standard and
unexplained
- No cross-validation or repeated runs — all results rest on a single
train/test split
- The Grad-CAM analysis, while suggestive, examines a single image under two
pipelines and cannot support the weight the discussion places on it

---
Major Comments

M1. Single Train/Test Split Without Repetition or Cross-Validation

All 65 pipeline evaluations use one fixed random seed for data splitting (line
  105). With no repeated runs, confidence intervals, or cross-validation, the
reported α values are point estimates whose sampling variability is unknown. A
  difference of 0.08 pp (N alone) could easily be noise. The entire variance
decomposition (ANOVA, η², F-tests) treats these single-run estimates as if
they were population parameters. This is the most serious methodological
concern.

Suggestion: At minimum, acknowledge this limitation explicitly. Ideally,
report standard deviations from 5–10 repeated runs with different random
seeds, or use k-fold cross-validation. If computational cost prohibits 65 × k
runs, at least repeat a representative subset (e.g., all length-3 pipelines)
to establish that the ordering-dominance finding is robust to seed choice.

M2. MSE as Loss Function for Binary Classification (line 105)

The paper uses mean squared error (MSE) as the loss function for a binary
classification task with sigmoid output. This is non-standard — binary
cross-entropy (BCE) is the conventional and theoretically motivated choice for
  this architecture. MSE with sigmoid outputs produces vanishing gradients when
  predictions are confident but wrong, potentially slowing convergence and
altering the performance landscape. Since the entire paper characterizes this
landscape, the loss function choice matters: the ordering sensitivities
observed may be partly artifacts of the MSE+sigmoid combination. This should
be justified or, at minimum, discussed as a limitation.

M3. Generalizability Limitations Need Stronger Framing

The paper's title and abstract make broad claims ("Image Preprocessing for
Skin Disease Classification") but the experimental evidence comes from:
- One custom 4-block CNN (no standard architectures like ResNet, EfficientNet,
  or ViT)
- One binary task (healthy vs. diseased, not even a clinically standard task
like melanoma vs. benign)
- One dataset combination (HAM10000 + Healthy Skin)
- One set of preprocessing parameters per transform

The Limitations section (V-E) acknowledges these individually, but the
abstract and conclusion should be more carefully scoped. As written, a reader
could reasonably believe these findings transfer to, say, ResNet-50 on
multi-class ISIC classification — which has not been established.

M4. The Baseline Accuracy of 97.1% Is Suspiciously High and Underexamined

A custom 4-block CNN achieving 97.1% on binary classification of dermoscopic
images warrants examination. The HAM10000 dataset contains 10,015 images
across 7 lesion categories; combining these into a single "diseased" class and
  pairing with a separate "healthy skin" dataset creates a task that may be
substantially easier than standard dermoscopic classification. The paper does
not report:
- Class balance (diseased vs. healthy proportions)
- Whether HAM10000's inherent class imbalance was addressed
- Absolute test-set size
- Whether the "Healthy Skin" dataset is truly independent of HAM10000's
acquisition protocol

At 97.1% baseline, the dynamic range for α is asymmetric (~3 pp above, ~97 pp
below), which may bias the length-degradation finding: it is structurally
easier to degrade performance from near-ceiling than to improve it. This
ceiling effect should be discussed.

M5. Variance Decomposition Interpretation at Length 3

The ANOVA at length 3 reports η² = 0.224 for between-set (selection) with
F(3,20) = 1.92 and p = 0.158, meaning the between-set effect is not 
statistically significant at any conventional threshold. The paper correctly
reports the numbers but then interprets them as if the 77.6% within-set
variance is a definitive finding. With only four groups of six observations
each, the ANOVA is underpowered to detect moderate selection effects. The
non-significant F-test means we cannot reject the null that selection has zero
  effect at length 3 — which is different from concluding that ordering
dominates. This distinction should be made explicit.

---
Minor Comments

m1. Title Length (line 36)
The title is 23 words. IEEE TIP guidelines typically recommend titles under 12
  words. Consider condensing, e.g., "Transform Ordering Dominates Selection in
Image Preprocessing Pipelines for CNN-Based Skin Disease Classification."

m2. \markboth Header Claim (line 43)
The running header reads "IEEE Transactions on Image Processing" — this is
appropriate only if the paper has been accepted. For a submission, this should
  be removed or replaced with a generic header.

m3. ReLU Citation (line 103)
The citation ramachandran2017searchingactivationfunctions is about searching
for activation functions; ReLU itself dates to at least Nair & Hinton (2010).
If the intention is to cite ReLU, the reference is incorrect. If the intention
  is to cite the architecture search that validated ReLU, that should be made
explicit.

m4. "Mean squared error" for Classification (line 105)
Beyond the major concern (M2), the notation is potentially confusing because
MSE is the standard abbreviation for a regression loss. If this choice was
deliberate, label it as such and explain briefly why.

m5. Permutation Formula (line 140)
The formula f(x) = Σ_{k=1}^{x} Π_{i=0}^{k-1} (x−i) computes the total number
of permutations across all lengths. While correct, it would be clearer to
simply state: "This yields $\sum_{k=1}^{4} P(4,k) = 4 + 12 + 24 + 24 = 64$
non-baseline configurations." The product-sigma notation obscures rather than
illuminates for most readers of an image processing journal.

m6. Normalization Equation (line 112)
The formula x' = (x − μ)/σ uses μ and σ, but the parameters table says Mean:
0.4, Std. dev.: 0.2. It is unclear whether μ and σ refer to the image
statistics (z-score standardization) or the target parameters (affine
rescaling to target mean/std). The text says "toward a target mean μ_t and
standard deviation σ_t" but the equation uses μ and σ without subscripts.
Clarify the equation.

m7. "Color Space Conversion (CS) transforms the image from RGB to HSV" (line 
118)
This is a lossy information transformation only if downstream processing
discards channels. Does the CNN receive all three HSV channels? If so, the
conversion is invertible and the claim that it "separates luminance from
chrominance" enabling independent channel exploitation should be qualified —
the network can learn equivalent representations from RGB.

m8. Preprocessing Parameter Selection (lines 120–121)
"Preprocessing parameters were determined through systematic grid search
optimization" — this grid search is not described. How many parameter
combinations were evaluated? Was the grid search conducted on the same data
partition as the main experiments? If so, there is a risk of information
leakage from the test set into the parameter selection process.

m9. "Approximately 23% of pipelines achieve positive α" (line 196)
By my count: 3 (length 1) + 5 (length 2) + 5 (length 3) + 2 (length 4) = 15 of
  64 = 23.4%. The word "approximately" is fine, but it would be clearer to say
"15 of 64 (23.4%)."

m10. Figure 1 Readability
The box plot (fig_length_effect) is very small when rendered at single-column
width. The percentage annotations ("75%+", "42%+", etc.) and the statistical
annotation box are in 6pt font — this may be below the legibility threshold
after print reproduction. Consider increasing font sizes or panel height.

m11. Figure 4 (Grad-CAM) Caption (line 321)
The caption describes panel (d) as showing a "structured negative-class
activation around the lesion." Inspecting the image, the activation ring in
panel (d) is visible but the description "ring structure" is interpretive.
Consider more neutral language like "concentrated negative-class activation in
  the lesion region."

m12. No Data Availability Statement
The paper does not state whether the trained models, preprocessing code, or
results data are publicly available. IEEE TIP encourages reproducibility; a
data/code availability statement should be added.

m13. "CUDA 13.2" (line 105)
As of early 2026, NVIDIA CUDA 13.2 does not exist (the latest stable release
is 12.x). Verify whether this should be CUDA 12.x or a different versioning
scheme (e.g., cuDNN version, driver version).

m14. Residual Terminological Inconsistency (lines 72, 357)
Line 72 refers to denoising as "approximately position-neutral" while line 357
  uses a three-level spectrum taxonomy ("strongly position-sensitive,"
"moderately position-sensitive," "position-neutral"). The spectrum model is
more precise — ensure the Introduction's language foreshadows rather than
contradicts it.

m15. The irjet Citation (line 112)
The reference for normalization is to Verma & Dutta (2017) in the
"International Research Journal of Engineering and Technology" — a journal
with limited peer review. Consider citing a more authoritative source for
intensity normalization (e.g., Gonzalez & Woods, already in the bibliography).

---
Questions for Authors

1. Was any data augmentation applied during training? If so, how does
augmentation interact with the preprocessing transforms? If not, why was
augmentation omitted for a relatively small dermoscopic dataset?
2. What is the class balance (healthy vs. diseased images) in the training,
validation, and test sets? At 97.1% baseline, is the model simply predicting
the majority class for most examples?
3. Were the preprocessing parameters (Table I) optimized independently for
each pipeline configuration, or were they fixed once and reused across all 65
pipelines? If fixed, the "optimal" parameters for equalization in isolation
may be suboptimal when equalization follows normalization — which would
amplify ordering effects beyond what would occur with per-pipeline parameter
tuning.
4. The Grad-CAM analysis uses images at confidence 1.0. Since both models are
equally and maximally confident, what additional information does the
activation map comparison provide about classification quality? Could you
repeat the comparison for a decision-boundary image (confidence ≈ 0.5) where
ordering effects might be more visible in the prediction itself?
5. The cost-adjusted metric αw (Eq. 3) uses an asymmetric formulation that is
not standard. What is the justification for dividing by γ when α ≥ 0 but
multiplying by γ when α < 0? Has this metric been used or validated elsewhere?
